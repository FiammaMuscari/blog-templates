# ¿Qué es un sdk?


Un SDK (Software Development Kit 🧐) es un conjunto de herramientas que ayudan a los desarrolladores a crear aplicaciones para una plataforma específica, incluye libs de código, herramientas de desarrollo, examples, tests y documentación. Contiene todos los componentes que un desarrollador necesita para crear nuevas aplicaciones para un producto específico y su entorno.

Suelen ser integraciones rápidas de servicios de aplicaciones como el pago (<a href="https://www.mercadopago.com.ar/developers/es/docs/sdks-library/landing">Mercado Pago</a>), la ubicación (<a href="https://developers.google.com/maps/documentation/android-sdk/overview?hl=es-419"> google maps</a>), los mensajes (<a href="https://chatsdk.co/"> Chatsdk</a>), el análisis y la publicidad (<a href="https://cloud.google.com/appengine/downloads?hl=es-419">Google Cloud sdk</a>). Por otro lado algunos SDK ayudan a crear software específico en una plataforma, como sucede con el desarrollo de aplicaciones Java, los queridos programadores que hayan usado este lindo lenguaje bien sabrán que se tarda más en configurar el IDE que en hacer un hola mundo, gracias al famoso <a href="https://www.oracle.com/java/technologies/downloads/">JDK</a> que puede descargarse en la página oficial de Oracle, este kit de desarrollo es vital para programar en este lenguaje. ¿capisci?

# What is an sdk?


An SDK (Software Development Kit 🧐) is a set of tools that help developers build applications for a specific platform. It includes code libs, development tools, examples, tests, and documentation. It contains all the components that a developer needs to build new apps for a specific product and its environment.

They are usually fast integrations for application services such as payment (<a href="https://www.mercadopago.com.ar/developers/es/docs/sdks-library/landing">Mercado Pago</a>), location (<a href="https://developers.google.com/maps/documentation/android-sdk/overview?hl=es-419">Google maps</a>), messaging (<a href="https://chatsdk.co/">Chatsdk</a>), analytics and advertising (<a href="https://cloud.google.com/appengine/downloads?hl=es-419">Google Cloud sdk</a>). On the other hand, some SDKs help to build specific software natively, as happens with Java application development; beloved programmers who have used this nice language will know well that it takes longer to configure the IDE than to print a hello world, thanks to the famous <a href="https://www.oracle.com/java/technologies/downloads/">JDK</a> that can be downloaded from Oracle's official page; this development kit is vital to program in this language. Well, let's recalculate, which one do I use then? Look, an SDK offers complete tools to build applications quickly, it works great for enterprise companies; meanwhile an API facilitates communication between platforms, it allows flexible and efficient integrations but usually requires more work and love.
Generally, both are used, since an SDK usually comes with an API. The choice between API and SDK depends on the specific needs of the project and the preference of the programmer leading the work.
APIs are more common in simple integrations, while SDKs are preferred for more complex projects.

Still, you're mostly likely gonna use APIs for now, but I want you to know that you can easily use an sdk like Mercado Pago's ones; where even though the API exists, there are also these integrations called sdk both for the Server Side (i.e. the backend adapted for PHP, Java, Node, Ruby, Net or Python) and for the Client Side (the frontend, with React or modules).
